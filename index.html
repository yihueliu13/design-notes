三層各自的命名範例
ref ref / color / blue / 500
→ 只有 Type + Role，後面接的是色彩名稱和色階，不涉及任何元件
sys sys / color / primary
→ Type + Role + 語意角色，描述「用途」。不知道也不在意它實際是什麼顏色
comp comp / color / button / filled / fill / hover
→ 完整七層漏斗，精確到元件的特定樣式、部位、狀態

關鍵觀念：選了 ref 就只會用到前幾層（Type + Role + 色彩值），因為原始值不屬於任何元件。選了 sys 會多一層語意角色和可選的 Target。只有選 comp 時才需要填完整的 Component → Variants → Target → Status。Plugin 面板中選 null 就是跳過該層。
